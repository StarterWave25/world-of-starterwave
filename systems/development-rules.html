HTML Development Rules CSS Development Rules

  • Why you need to follow the Development Rules?

    These Rules align you in StarterWave to Follow the Same Code Standards.

    They produce Readable Code. So anyone in StarterWave can Understand your Code easily because the code looks like they had wrote it.

    And also Debugging of the Code becomes Easier.

  • HTML Development Rules:

    1. Element Structure in Instruction Docs:

    In the Task Instruction Docs, the HTML Structure of the Page is Written by following Structure:

    This Structure helps you to Understand How to Arrange and Name the Elements in HTML Code.

    ( div > h3 ) means, h3 Element is a Direct Child to div Element i.e h3 is Placed inside div.

    ( h3 + div ) means, div Element is a Direct Sibling to h3 Element i.e div is Placed next to h3.

    h3(.product-title) means, h3 with Class Attribute Value as product-title.

    h3[Product Title] means, h3 should have the Content inside as Product Title.

    Ex: body > header + main + footer

    Ex: div(.resume-templates-container) > h2 + div (.resume-templates-grid)

    2. Every HTML Page must have Semantic Elements, the Page Structure Follows:

    header > nav > ul > li > a

    main > section (Don't use nested section, use div instead)

    footer > nav > ul > li > a

    3. To apply Styles to an Element, use Class Attributes to Select the Element in CSS.

    Using Class Attribute for naming an Element maintains Equal Specificity through out the CSS Code.

    It helps to Stop Overriding Unnecessary Styles of an Element.

    4. Write Purposeful Class names of an Element, Even if they are Long.

    If Class Names are Meaningful then we can Easily Identify the Styles of an Element in CSS File.

    Class Names must be written in kebab-case.

    Bad Class Names: prods-cont, prod-card, abc, pc, inputField, table, btn, card-div.

    Good Class Names: products-container, product-card, input-field-item, product-title, product-price, orders-table, show-orders-btn.

    5. Every Webpage must have Only One H1 Heading Element.

  • CSS Development Rules:

    Always prefer to use Multiples of 8 Values for gap, padding, margin.

    Spacing will be Consistent, Looks Clean and Matches Industry Standards.

    Ex: gap: 8px (Correct), gap:6px (Not Good).

    Ex: gap: 2rem (Correct), gap: 1.7rem (Not Good)

    1. Select Every Element you use in CSS Code through Class Names Only except Single Unique Elements like body, header, footer, main, etc.

    2. Select all Elements remove Default Behaviours

    Apply them margin & padding as 0.

    Change its box-sizing property value to border-box. To Change the Width & Height Calculation Behaviour of Elements.

    3. Use Flexible Sizes Instead of Fixed Sizes

    Don’t use fixed sizes like px everywhere. Use flexible values like %, vw, or clamp().

    Your design will adjust automatically on mobile, tablet, and desktop.

    4. Use Flexbox to Arrange Items in (Rows/Columns).

    It makes alignment and spacing very easy compared to other methods.

    5. Use Consistent Gap for Flex Container instead of Margin Between Flex Items.

    Easier and cleaner than adding margin to each element.

    6. Always Add Consistent Padding or Space Inside all Sections in the Webpage.

    Without space, everything looks crowded and messy.

    7. Stack Items on Small Screens

    Use media query to change Flex Container Layout to column on mobile.

    Side-by-side layouts don’t fit on small screens.

    8. Allow Flex Items to Wrap automatically

    Use flex-wrap: wrap. Items will automatically move to next line on small screens.

    9. Make Text Size Flexible

    Use clamp() for font sizes. Text becomes readable on all devices without extra work.

    10. Keep Images Properly Sized

    Use width: 100% and object-fit: cover. Images won’t stretch or break layout.
